---
title: "CS2 WebRadar architecture: backend, relay, and frontend"
description: "How the C++ backend, Node.js WebSocket relay on port 22006, and Vite + React 18 frontend cooperate to deliver live CS2 game data to browsers."
---

CS2 WebRadar is built from three cooperating components: a C++ backend that reads live game state from the CS2 process, a Node.js WebSocket relay that fans messages out to connected browsers, and a React frontend that renders the live radar. Data flows in one direction — from CS2 memory to every watching browser — with no round-trips required after the initial WebSocket handshake.

## Data flow

```
CS2 process
    │  (process memory)
    ▼
cs2_webradar.exe   (C++ backend)
    │  WebSocket message — binary JSON blob
    ▼
Node.js relay   ws://host:22006/cs2_webradar
    │  broadcast to all connected clients
    ▼
Browser(s)   React frontend
```

Each time the C++ backend produces a frame it sends a single binary WebSocket message to the relay. The relay immediately forwards that message to every connected browser client. Browsers parse the JSON payload and update React state, triggering a re-render of the radar, player cards, scoreboard, and all overlays.

### C++ backend

The executable (`cs2_webradar.exe`) runs alongside CS2 and uses the game's own SDK and memory offsets to read live entity state. On each tick it serializes the following into a single JSON object and sends it as a binary WebSocket message to the relay:

- `m_players` — array of all player controller + pawn data
- `m_bomb` — C4 position and timer state
- `m_grenades` — all active grenade projectiles and inferno entities
- `m_round_info` — game rules (round number, phase, timer, scores)
- `m_map` — the active map name string (e.g. `"de_dust2"`)
- `m_local_team` — the team integer of the local player (`2`=T, `3`=CT)

### Node.js relay

The relay (`webapp/ws/app.js`) is intentionally minimal. It does no parsing, filtering, or transformation — it simply re-sends every inbound message to every currently connected client:

```js
web_socket_server.on("connection", (web_socket, request) => {
  web_socket.on("message", (message) => {
    web_socket_server.clients.forEach((client) => {
      client.send(message);
    });
  });
});

server.listen(port);
```

This fan-out relay pattern means any number of browsers can watch the same game simultaneously. The C++ backend only ever manages one outbound connection.

### React frontend

The frontend (`webapp/src/`) is a Vite + React 18 SPA styled with TailwindCSS. The root component (`app.jsx`) owns all WebSocket state and distributes data down to:

| Component | Responsibility |
|---|---|
| `Radar` | Renders the map image and all entity overlays; handles zoom, pan, and fullscreen |
| `Player` | Positions a player dot on the radar; rotates with `m_eye_angle`; shows view cone |
| `Bomb` | Positions the C4 icon using world coordinates |
| `Grenade` | Renders grenade dots, smoke clouds, and fire effects |
| `PlayerCard` | Sidebar card with health, weapons, money, KDA, and armor |
| `Scoreboard` | Round timer, team scores, and bomb countdown bar |
| `KillFeed` | Scrolling log of kills derived from player state changes |
