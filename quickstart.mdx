---
title: "Quickstart: install and run CS2 WebRadar on Windows"
description: "Install the web dependencies, start the C++ backend and React interface, and open the live radar in your browser in under five minutes on Windows."
---

CS2 WebRadar requires two things running at the same time: the C++ backend that reads CS2 memory, and the React web interface that visualises the data. This guide walks you through both.

## Prerequisites

- Windows 10 or 11
- Counter-Strike 2 installed via Steam
- Node.js 18 or later installed ([nodejs.org](https://nodejs.org))
- The CS2 WebRadar release downloaded and extracted ([GitHub releases](https://github.com/Kava4/cs2-webradar/releases))

<Steps>
  <Step title="Install web dependencies">
    Open the folder where you extracted CS2 WebRadar and double-click **`install.bat`**, or run it from a terminal.

    ```bat install.bat
    @echo off
    start "cs2_webradar - webapp" cmd /k "if exist "webapp" (echo [cs2_webradar] installing webapp dependencies.. && cd "webapp" && npm install && npm audit fix && echo [cs2_webradar] installed react dependencies) else (echo [cs2_webradar] failed to install webapp dependencies! make sure you followed the instructions carefully && pause)"
    ```

    This changes into the `webapp/` directory and runs `npm install` to download React, Vite, and all other frontend dependencies. Wait until you see `[cs2_webradar] installed react dependencies` in the console window before continuing.
  </Step>

  <Step title="Start Counter-Strike 2">
    Launch CS2 through Steam and load into a match, practice server, or offline bot game. The backend must be able to attach to a running CS2 process, so the game needs to be open before you start the executable.
  </Step>

  <Step title="Run the backend">
    Double-click **`cs2_webradar.exe`** in the release folder. The executable reads CS2 process memory and starts a WebSocket server on port `22006`.

    <Note>
      On first run, Windows Firewall will ask you to allow network access. Click **Allow access** to let the WebSocket server communicate on your local network. If you skip or cancel this prompt, the web interface may fail to connect. You can re-add the exception later in **Windows Defender Firewall → Allow an app through firewall**.
    </Note>

    A `config.json` file is created in the same folder on first run. See [configuration](/configuration) for details on what each field does.
  </Step>

  <Step title="Launch the web interface">
    Double-click **`start.bat`**, or run it from a terminal.

    ```bat start.bat
    @echo off
    start "cs2_webradar - webapp" cmd /k "if exist "webapp" (cd webapp && npm run dev) else (echo [cs2_webradar] failed to start webapp! make sure you followed the instructions carefully && pause)"
    ```

    This runs `npm run dev` inside `webapp/`, which starts both the Vite development server and the Node.js WebSocket relay (`ws/app.js`) using `concurrently`.
  </Step>

  <Step title="Open the radar in your browser">
    Navigate to **[http://localhost:5173](http://localhost:5173)** in any browser. The radar map appears and connects to the backend automatically. You should see live player positions update as the game runs.

    <Tip>
      If you see a "Connecting…" spinner for more than a few seconds, make sure `cs2_webradar.exe` is still running and that the Windows Firewall prompt was accepted. Restarting both the backend and `start.bat` usually resolves connection issues.
    </Tip>
  </Step>
</Steps>

## Sharing the radar with a friend

To let a remote friend watch your game in their browser, forward TCP port `22006` on your router to your PC's local IP address. Then click **Copy Share Link** inside the radar UI — it generates a URL that encodes your public IP, which you can send to your friend. See [sharing the radar](/usage/sharing) for the full walkthrough.
